About the App
NSWeXplorer Data API is the data interface used by NSWeXplorer for Apps4NSW contest. The API has implemented by Mingyi Wang and Zifei Gong.

The application integrates lots of datasets such as attractions, services and acitivities information and having the ability to interact with further datasets. It converts the raw data from different data sources with different format into a json responce which contains geographic information and other details for different kinds of facilities in NSW state. It also exposes the interface by returning a WSDL file, which allows developers to further consume the service. Our other application NSWeXplorer can be seen as a prototype for the usage of this API.

Why the App
The main purpose of having this API is to help developer who has great ideas to overcome the problem of integrate different data sources, which usually the biggest time consuming phase when making a mashup. By using the API, it should easier the development and allows people do consentrate on their ideas and business logic instead of spending too much time on data layer.
About the Data
  • Australia Post’s postcode data (link)
  • National Public Toilet Data (link)
  • Reserves & Parks data fetched from local councils, such as Randwick, Bankstown, Campbelltown, etc
  • Sport clubs and facilities data fetched from local councils, such as Bankstown, Parramatta, Campbelltown, etc
  • Other data fetched from Yellow Pages under non-commercial use conditions. (Yellow pages’ data license)

Some data are fetched from web pages, we use them just for an demonstration purpose for our idea, with more data provided by Australian Government, further attractions and activities can be added to the Application similarly.

The challenges we faced
The biggest challange we faced was integrating different format of data. The data are from Government departments, Local councils and web pages. Most of them was in Excel documents and a few of them are XML files, the rest are displayed on web pages. Those data are strctured for human reading but not for computer understanding. So having a unified interface which has the ability to integrate them as well as further datasets is important and can save huge time in future development.

Besides the data format, we also faced a problem which is there are actually a quite limited range of data available for implement our whole idea. Some data we found has not including NSW states as well. Although there're quite a few statistic datasets which can be obtained from Australian Bureau of Statistics, we are not satisfied just implement another mashup which only shows statistics like Crime rate, population, etc, since there're already something like that, e.g. Know Where You Live, Suburban Trends. However NSWeXplorer is just a prototype for our idea, and all rights of the data belongs to its data provider.

Finding a stable host is also a little bit time consuming. Since we are using J2EE technology, which we though more flexible and powerful then PHP, find a cheap and reliable hosting needs a little bit more work. We are currently hosting our mashup on Stax.net, which is an Cloudy platform based on Amazon EC2. Currently the site can not afford very high traffic or requests, and there are might be some server downtime for maintainance.
